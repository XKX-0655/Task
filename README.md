# Task

Task 是一个给自己用的 AI 授权自托管自律系统。它把任务、日记、身体状态、学习记录、AI 对话和奖惩监督放在一起，让用户通过手机端授权 AI 读取 App 内数据，并在规则范围内自动执行奖励和惩罚。

第一版仓库只写产品方案和路线图，不制作 APK、不创建 Android 工程。目标是先把 App 应该是什么样、未来怎么做清楚。

## 核心想法

- 用户可以在 App 里创建任务、写日记、记录身体状态、记录学习情况。
- App 会按阶段申请手机权限，权限必须通过 Android 系统和 App 内设置显式授权。
- 用户可以在 App 里配置 AI API、模型和 prompt，并和 AI 助手对话。
- AI 助手可以读取用户写入并授权给它的 App 内数据，包括任务、日记、身体状态和学习记录。
- 用户先设置 AI 能做什么奖励和惩罚；授权后，AI 可以在这个范围内自托管执行，不需要每次确认。
- 第一阶段奖惩以数字积分为核心，后续可以扩展到页面内容、提醒强度、顶部提示或悬浮干扰。

## 权限与安全边界

Task 的权限分为三层：

- 基础权限：本地数据、通知、AI API 配置。
- 监督权限：定时提醒、顶部提示、悬浮提醒、任务失败处理。
- 深度权限：无障碍、使用情况访问、设备管理等，后续单独研究。

App 不应该静默获取权限，不应该绕过系统授权，也不应该实现无法关闭、无法退出、无法卸载的机制。
